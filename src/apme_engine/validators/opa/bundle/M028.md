## first_found auto-splitting paths (M028)

`first_found` lookup auto-splitting paths on delimiters is deprecated (2.23).

**Removal version**: 2.23
**Fix tier**: 1
**Audience**: content

### Detection

Detects `first_found` lookup terms containing comma or colon delimiters that
trigger the deprecated auto-splitting behavior. The `first_found` lookup is
typically invoked inside Jinja2 expressions, which the OPA hierarchy does not
decompose into `module_options`; these examples cannot be tested through the
integration harness. OPA unit tests in `M028_test.rego` validate the Rego
logic directly.

### Example: violation

```yaml
- name: Include first found vars
  ansible.builtin.include_vars: "{{ lookup('ansible.builtin.first_found', 'a.yml,b.yml') }}"
```

### Example: pass

```yaml
- name: Include first found vars
  ansible.builtin.include_vars: "{{ lookup('ansible.builtin.first_found', ['a.yml', 'b.yml']) }}"
```

### Remediation

Split the comma-delimited string into a YAML list.
