## Debug sensitive variables (L110)

Debug tasks should not log sensitive variables without no_log: true.

Detects debug module usage where msg or var contains references to sensitive
variables (password, secret, token, api_key, credential, etc.) without no_log
protection on the task or an ancestor scope (block, play).

### Example: violation

```yaml
- name: Show database password
  ansible.builtin.debug:
    msg: "Password is {{ db_password }}"
```

```yaml
- name: Display API credentials
  ansible.builtin.debug:
    var: api_token
```

### Example: pass

```yaml
- name: Show database password (protected)
  ansible.builtin.debug:
    msg: "Password is {{ db_password }}"
  no_log: true
```

```yaml
- block:
    - name: Show credentials
      ansible.builtin.debug:
        var: api_secret
  no_log: true
```
