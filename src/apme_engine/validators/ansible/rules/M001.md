## FQCN resolution (M001)

Uses `ansible.plugins.loader.module_loader.find_plugin_with_context()` to resolve a module name against the installed ansible-core and collections. Reports when a short or non-canonical module name resolves to a different FQCN, suggesting the canonical form be used instead.

### Example: violation

```yaml
- name: Example play
  hosts: localhost
  connection: local
  tasks:
    - name: Uses short name
      yum:
        name: httpd
        state: present
```

### Example: pass

```yaml
- name: Example play
  hosts: localhost
  connection: local
  tasks:
    - name: Uses FQCN
      ansible.builtin.dnf:
        name: httpd
        state: present
```
